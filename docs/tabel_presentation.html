Tabel Motion Web

Система табеля, оценок и отчётности для учебного центра

Веб-платформа для админа, ментора и студента: управление группами, месячный табель, журнал успеваемости и автоматическая отправка отчётов через Dify.

3 роли Админ, ментор, студент
1 табель Месячная матрица по дням
React + DRF Современный web-стек
Dify Интеграция для отчётов и WhatsApp

Ценность проекта

Проблема, которую решает система

Учебным центрам сложно вести оценки, посещаемость и обратную связь для родителей в одном интерфейсе. Tabel объединяет весь этот процесс в единую платформу.

Платформа

Что уже реализовано в системе

Роли и доступы

Админ управляет группами, студентами и менторами. Ментор работает только со своими группами. Студент видит только собственные оценки и сводку.

Месячный табель

Табель открывается на весь месяц сразу. Оценки ставятся по дням, по каждому студенту, с подсчётом посещаемости и среднего балла.

Автоотчёты

В конце месяца система собирает персональный отчёт по студенту и отправляет его в Dify, где дальше можно строить доставку в WhatsApp.

Ключевые функции

1
Авторизация по ролям

JWT-вход, разграничение прав и персональные кабинеты для всех пользователей.

2
Группы и учебные дни

Создание групп, расписание по дням и быстрый доступ к табелю по каждому потоку.

3
Отчётность по студенту

Сбор оценок, пропусков, среднего балла и месячной статистики для дальнейшей отправки.

Что получает учебное заведение

Слайд 5 Mobile-friendly web version

Architecture

Техническая архитектура проекта

Стек

Frontend

React, Vite, CSS, адаптивный интерфейс, role-based страницы и матрица табеля.

Backend

Django, Django REST Framework, JWT, отчёты, API для ролей и табеля.

Данные

SQLite локально, возможность перехода на PostgreSQL для production-развёртывания.

Интеграции

Dify Workflow API для отправки месячных отчётов и дальнейшей доставки в WhatsApp.

Поток данных

1. Admin / Mentor
создаёт группу, уроки и оценки
↓
2. Django API
сохраняет данные табеля и журнал уроков
↓
3. Student
видит только свои оценки и месячную сводку
↓
4. Monthly Report
в последний учебный день месяца строится персональный отчёт
↓
5. Dify
принимает payload и выполняет сценарий доставки

Формат продажи

Проект можно предлагать как готовое MVP для учебных центров и IT-школ в Бишкеке.

Модель внедрения

Коммерческий потенциал

Подходит как SaaS/MVP для цифровизации учебного процесса и связи с родителями.
