# AGENTS.md

## О проекте

Tabel - учебная система на Django REST Framework и React/Vite для управления группами, менторами, студентами, табелем, посещаемостью и ежемесячными отчетами.

## Главные цели проекта

- Роли `ADMIN`, `MENTOR` и `STUDENT` должны иметь строгие и понятные права доступа.
- Администратор управляет менторами, студентами, группами и составом групп.
- Ментор ведет табель только по своим группам.
- Студент видит только свои оценки, посещаемость и краткую сводку.
- Ежемесячный отчет формируется по данным табеля и отправляется через Dify.

## Правило ежемесячного отчета родителям

Отчет по каждому студенту отправляется на номер родителя, а не студенту. Номер берется из `StudentProfile.parent_phone`.

Когда отправлять:

- Только в день последнего урока группы за отчетный месяц.
- Последний урок месяца определяется по максимальной `Lesson.lesson_date` внутри этого месяца.
- Отчет отправляется только после того, как у студента появилась запись оценки или пропуска на этом последнем уроке.
- В начале следующего месяца отчет за прошлый месяц автоматически отправляться не должен.
- Один и тот же отчет за пару `(student, month)` нельзя отправлять несколько раз. Если успешная отправка уже записана в `MonthlyStudentReportDispatch`, повторная отправка запрещена.

Где находится логика:

- `tabel_project/tabel_app/report.py` - расчет последнего урока, сбор payload, защита от ранней и повторной отправки, отправка в Dify.
- `tabel_project/tabel_app/views.py` - после сохранения табеля проверяет, не пора ли отправить отчет студентам.
- `tabel_project/tabel_app/models.py` - `MonthlyStudentReportDispatch` хранит историю и защищает от дублей.
- `tabel_project/tabel_app/management/commands/send_monthly_reports.py` - ручной запуск с сервера.

## Важные файлы

- `tabel_project/tabel_app/models.py` - пользователи, профили, группы, уроки, оценки, история отправки отчетов.
- `tabel_project/tabel_app/report.py` - месячные отчеты и интеграция с Dify.
- `tabel_project/tabel_app/scheduler.py` - опциональная автоматическая проверка отчетов.
- `tabel_project/tabel_app/views.py` - API и права доступа.
- `frontend/src` - клиентская часть React.
- `README.md` - публичное описание проекта, должно быть на русском и в нормальной UTF-8 кодировке.

## Проверки

- Backend: `python tabel_project/manage.py test tabel_app`
- Frontend lint: `cd frontend && npm run lint`
- Frontend build: `cd frontend && npm run build`

## Известные риски

- Русский текст в README и части интерфейса выглядит поврежденным из-за кодировки. Его нужно восстановить в нормальный UTF-8.
- `.env.save` похож на файл с реальными секретами. Нельзя хранить рабочие ключи, пароли и Dify API key в репозитории.
- Локальный запуск требует установленных Python-зависимостей и Node/npm.
